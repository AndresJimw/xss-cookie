Welcome to the XSS Cookie Lab

A deliberately vulnerable web application to study cross-site scripting (XSS), cookie theft, blind XSS and simple context-based mitigation.

Scenarios

Each endpoint represents a different attack surface along the XSS life cycle.

  • Reflected
    /search

    Reflects user input directly in the response to illustrate classic reflected XSS using a query parameter.

  • Stored
    /comments

    Persists comments on the server and renders them for every visitor, simulating a stored XSS vulnerability.

  • Blind
    /contact → /admin/messages

    Models blind XSS where the payload is only rendered when an admin opens the messages panel.

  • Impact
    /steal → /admin/cookies

    Captures cookie-like data sent to the collector endpoint and exposes it in an admin view to show tangible impact.

How to use

  1. Start with /search to observe a simple reflected XSS and how the mitigated view differs from the vulnerable one.
  2. Move to /comments to store payloads on the server and see how stored XSS affects every visitor to the page.
  3. Use /contact and then /admin/messages to simulate a blind XSS targeting an administrator who reviews user messages.
  4. Trigger requests to /steal from your payloads and review the collected evidence inside /admin/cookies.
  5. Switch the security mode to log or block using environment variables and compare the behavior of the vulnerable and mitigated views.
